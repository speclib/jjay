# A README Worth Reading

Thirty seconds. That's all you get to understand jjay.

Every project needs a front door, so I made the flock build one. Hero image up top, the tagline — *Control the flock.* — and then the part that matters: the manual workflow. Spawn a workspace, launch an agent, test, archive, merge, cleanup. Do that by hand for every agent and you'll go mad. jjay is the automation layer that does it for you. Name the problem, then name the fix.

Prerequisites are opinionated and non-negotiable: jj, tmux, openspec. Hard dependencies, no apologies. Installation says "coming soon" because it is. A rough sketch of the planned commands and a roadmap round it out.

No badges, no philosophy essay, no fake marketing polish. This is an alpha README that says what jjay does and gets out of the way. The hero image and the workflow do the talking.

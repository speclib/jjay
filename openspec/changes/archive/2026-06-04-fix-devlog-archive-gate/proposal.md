## Why

The devlog/blog artifact in the OpenSpec schema is never created because the propose workflow stops once `apply.requires: [tasks]` is satisfied, and the archive workflow only warns about missing artifacts without creating them. This means every change gets archived without a devlog entry, defeating the purpose of having the blog artifact in the schema.

## What Changes

- Extend the project's `CLAUDE.md` "OpenSpec Archive triggers" section so that `/opsx:archive` creates a missing `blog` artifact before archiving, rather than just warning about it
- The directive: when `/opsx:archive` runs, if the `blog` artifact is not `done`, automatically create it using `openspec instructions blog --change "<name>" --json` before proceeding — turning the soft warning into a gate with auto-resolution
- The fix lives in `CLAUDE.md`, **not** in `.claude/commands/opsx/archive.md`, because that file is generated by the OpenSpec CLI (regenerated on `openspec update`) and any local edit would be overwritten. `CLAUDE.md` already augments archive behavior (jj describe, changelog) via this same section, so this is a consistent, durable place for the directive.

## Capabilities

### New Capabilities
- `archive-devlog-gate`: The archive workflow creates missing blog artifacts before archiving instead of silently skipping them

## Impact

- **`CLAUDE.md`**: "OpenSpec Archive triggers" section gains a directive to auto-create a missing `blog` artifact before archiving
- **No code changes**: This is a prompt/instruction change to the archive workflow, not application code
- **Durable across OpenSpec updates**: Because the directive lives in `CLAUDE.md` (not the CLI-generated `archive.md`), it survives `openspec update`
- **Backward compatible**: Changes that already have a blog artifact are unaffected; changes without one get one created automatically at archive time
